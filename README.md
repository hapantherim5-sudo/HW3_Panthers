# HW3_Panthers

# הוראות להרצת הפרויקט

## הרצה ללא מפתחות

לאור המייל שנשלח על ידי נעמי וצוות הקורס בנושא הקושי בשימוש במפתחות במחברת משותפת, התאמנו את הפרויקט כך שניתן יהיה להריץ אותו גם ללא מפתחות ל־Gemini ול־Firebase.

כאשר המחברת מופעלת ללא המפתחות:

* ממשק המשתמש של הפרויקט יעלה ויהיה ניתן להשתמש בחלקים שאינם תלויים בשירותים חיצוניים.
* פעולות ה־AI המבוססות על Gemini לא יהיו זמינות.
* המידע לא יישמר ב־Firebase.
* במקום Firebase, המערכת תשתמש באחסון זמני בזיכרון. מידע זה יימחק כאשר סביבת ההרצה של Google Colab תיסגר או תופעל מחדש.
* חלק מהתכונות לא יציגו את ההתנהגות המלאה של הפרויקט כפי שהיא מתקבלת בעת חיבור ל־Gemini ול־Firebase.

לצורך בדיקה מלאה של כל יכולות הפרויקט, ניתן להוסיף למחברת מפתח Gemini ופרטי התחברות ל־Firebase בהתאם להוראות הבאות.

---

## הוספת מפתח Gemini

1. היכנסו ל־Google AI Studio באמצעות חשבון Google.
2. עברו לעמוד יצירת מפתחות ה־API.
3. לחצו על האפשרות ליצירת מפתח API חדש.
4. בחרו פרויקט Google Cloud קיים או צרו פרויקט חדש.
5. העתיקו את מפתח ה־API שנוצר.
6. פתחו את המחברת ב־Google Colab.
7. לחצו על סמל המפתח **Secrets** הנמצא בסרגל השמאלי.
8. הוסיפו Secret חדש בשם:

```text
GEMINI_API_KEY
```

9. הדביקו בשדה הערך את מפתח ה־API שהועתק מ־Google AI Studio.
10. הפעילו את האפשרות המאפשרת למחברת גישה ל־Secret.
11. הפעילו מחדש את תאי המחברת.

לאחר הוספת המפתח, הפעולות בפרויקט המשתמשות ב־Gemini יהיו זמינות. מפתח Gemini נוצר דרך Google AI Studio ומשמש לאימות הבקשות הנשלחות ל־Gemini API.

---

## הוספת פרטי Firebase

לצורך החיבור ל־Firebase יש להשתמש בקובץ Service Account בפורמט JSON.

1. היכנסו ל־Firebase Console.
2. פתחו את הפרויקט שבו תרצו להשתמש.
3. לחצו על סמל גלגל השיניים ובחרו **Project settings**.
4. עברו ללשונית **Service accounts**.
5. לחצו על **Generate new private key**.
6. אשרו את יצירת המפתח.
7. למחשב יורד קובץ JSON המכיל את פרטי ההתחברות של חשבון השירות.
8. פתחו את קובץ ה־JSON באמצעות עורך טקסט.
9. העתיקו את כל תוכן הקובץ, כולל הסוגריים המסולסלים בתחילתו ובסופו.
10. פתחו את המחברת ב־Google Colab.
11. לחצו על סמל המפתח **Secrets** הנמצא בסרגל השמאלי.
12. הוסיפו Secret חדש בשם:

```text
FIREBASE_SERVICE_ACCOUNT
```

13. הדביקו בשדה הערך את כל תוכן קובץ ה־JSON.
14. הפעילו את האפשרות המאפשרת למחברת גישה ל־Secret.
15. הפעילו מחדש את תאי המחברת.

לאחר הוספת פרטי חשבון השירות, המערכת תתחבר ל־Firebase ותוכל לשמור ולטעון את נתוני הפרויקט. Firebase Admin SDK משתמש בפרטי Service Account כדי לאמת את המערכת ולאפשר לה לתקשר עם שירותי Firebase.

---

## שמירה על אבטחת המפתחות

מפתח Gemini וקובץ ה־Service Account של Firebase הם פרטים רגישים. אין לפרסם אותם בקוד ציבורי, ב־GitHub או בקובץ המחברת עצמו.

מומלץ להוסיף אותם רק דרך מערכת ה־Secrets של Google Colab. במקרה שמפתח נחשף, יש לבטל אותו וליצור מפתח חדש. קובץ ה־Service Account כולל מפתח פרטי ולכן יש לשמור עליו באופן מאובטח.

## סיכום

ניתן להריץ את הפרויקט גם ללא מפתחות, אך במצב זה תכונות ה־AI ושמירת הנתונים הקבועה ב־Firebase לא יהיו זמינות במלואן.

לבדיקה מלאה של הפרויקט יש להוסיף את שני ה־Secrets הבאים:

```text
GEMINI_API_KEY
FIREBASE_SERVICE_ACCOUNT
```
